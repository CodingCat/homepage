Bring BigData to Everyone

We are developing a user-friendly Machine Learning system, so that

  • You can analyze large volume of data efficiently, even you never learn programming stuffs

  • You can evaluate your new algorithms, even you are a machine learning expert but with limited system background

Automatic Optimization Engine

The optimization engine should takes the generated workflow of machine learning tasks as input, and determine:

  • Data layers for input and intermediate results. In machine learning algorithms, the intermediate results input should be stored in memory instead of permanent store for multiple-time using. Spark [1] optimizes large scale data processing performance by this approach, while it requires users to explicitly indicate which piece of data should be kept in memory. This scheme requires users to have not only rich experiences on programming and machine learning algorithms, but also the rich system experiences.
  • Search space tailoring. To determine the combination of multiple ML algorithms, parameters and features requires exploration in a huge space. A optimizer should consider materialized intermediate results (pre-processed data) as well as general statistics about the data to estimate the model learning time and the expected quality for each of the model configurations to focus on the most promising candidates.
